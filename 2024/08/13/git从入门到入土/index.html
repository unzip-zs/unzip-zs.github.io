git的复杂操作

指一些正常生产中常用的git操作集 * Commit message 和 Change log编写规范 原文参考阮老师的博客，点击前往原文 * 要点： * commit message * 基本要求 应该清晰明地说明本次提交的目的 * 格式 包括Header，Body 和 Footer部分 * 遵循应用广泛的Angular规范
* 编写插件： Commitizen * 使用git cz 代替git commit

  * Change log
    * Change Log 是发布新版本时，用来说明与上一个版本差异的文档
    * **自动生成插件，要求commit满足Angular规范：** conventional-changelog
